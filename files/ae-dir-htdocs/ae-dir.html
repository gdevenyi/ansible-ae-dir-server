Intro

Æ-DIR is an acronym for Authorized Entities Directory (and a small Unicode challenge), an LDAP service based on OpenLDAP's slapd.

Main design paradigms:

  • Explicit is better than implicit
  • Individual authentication required, no anonymous access at all
  • Strictly follow need-to-know-principle with fine-grained authorization
  • Avoid all-mighty proxy accounts
  • Delegated administration of manageable small areas
  • Do not assume hierarchical structure
  • A person is not an account
  • Role separation with multiple user accounts per person
  • Persistent IDs (never re-used) for meaningful audit trail
  • Only encrypted network traffic
  • Well-defined semantics for all object classes and attributes (better no data than bad data)

For a longer introduction you can check out various Æ-DIR conference presentations.
